## 1) Главная находка — почему ИИ «не работал»

**Факт с прода (17.07):** 9954 статьи, суть есть лишь у **3455** (~**6500 не обработаны никогда**),
очередь `external-ai` пуста, планировщик жив, конфиг корректен
(`limit=50 region=external ext=1 offline=0`, `restarts=0 oom=false`).

**Механизм.** На проде (внешний путь) гейт релевантности идёт **первым**, и отклонённой статье
суть **не пишется** — `external_ai.process_payload`: `if not relevant: continue`. А отбор в
обработку шёл по `WHERE c.summary IS NULL` **без фильтра по `relevant`**, поэтому отклонённая
статья вечно подходила под условие и возвращалась в очередь **каждый цикл**.

Гейт режет **~78%** (2696 отклонённых). Значит из 50 слотов цикла ~39 уходили на **повторное
отклонение одних и тех же статей** → они снова остаются без сути → снова в топе
(`ORDER BY published_at DESC`). Свежие вытеснялись. Самоусиливающийся капкан.

Побочно — каждый цикл заново жёг дорогой гейт (gpt-5.5) на том же наборе: это и есть «утечка
денег», зафиксированная аудитом 09.07. **Аудит был прав.**

**Фикс (PR #26):** `AND c.relevant IS NOT FALSE` в `get_articles_needing_summary`
(`repository.py`). Идиома уже принята в проекте (`list_articles`, `digest_candidates`,
`dashboard_stats`). Именно `IS NOT FALSE`, а не `IS NULL` — чтобы релевантная статья без сути
(сбой записи) попала на честный повтор.

**Признак, что фикс сработал** (через цикл-два после деплоя): в `external-ai` появляются задачи,
а «с сутью» растёт **на новых** статьях:
```bash
docker exec oiltech_app python -m oiltech_digest.cli external-queues-status
docker exec oiltech_pg psql -U oiltech -d oiltech_digest -c \
  "SELECT COUNT(*) FROM article_cards WHERE COALESCE(summary,'')<>'';"   # было 3455
```

## 2) T3 — мягкое удаление не работало (PR #25)

`get_background_job` делает `SELECT *`, поэтому ключи строки = имена колонок:
**`payload_json`** (`schema.sql:304`) и **`result_json`** (:305). Код читал несуществующие
`"payload"` / `"result"` и молча получал `{}`:

- `api.py` `external_worker_complete` → `force/dry_run/mark` **всегда False** → `apply_recheck_result`
  уходил в ветку по умолчанию и **удалял статьи физически**, даже когда просили пометить.
  Так уже потеряли ~2000 статей.
- `cli.py` `recheck-dry-show` → `applied` всегда `{}` → превью dry-run было недоступно.

**После деплоя задача №1 бэклога («перепрогон в мягком режиме») становится безопасной.**

## Важное / подводные камни

- **Локальный и внешний AI-пути имеют РАЗНЫЙ порядок стадий.** Локальный: суть → гейт (отклонённые
  получают суть, в ловушку не попадают). Внешний (**прод**): гейт → суть. Чтение локального кода
  даёт **ложный вывод**, что бага нет — на этом легко обжечься.
- **Моки скрывают баги.** И T3, и капкан отказников выжили потому, что тесты мокали задачу
  рукописным словарём (`{"id":…, "kind":…}`) или функцию целиком, а реальной формы строки/БД
  никогда не видели. Новые тесты повторяют реальную форму и **проверены откатом** (падают на баге).
- **`docker logs --since 24h` врёт после пересоздания контейнера.** Логи живут столько же, сколько
  контейнер. Сначала `docker ps` (возраст) — потом выводы.
- **Скрипт планировщика пишет `FAIL`, а не `error`** — grep по «error» молчит даже при падениях.
- **Спящая мина:** в ветке `STREAMING_PIPELINE=1` (`docker-scheduler.sh:78`) проверяется только
  наличие `OPENAI_API_KEY` и **не проверяется `AI_EXECUTION_REGION=external`**. На РФ ключа нет по
  дизайну → при включении стриминга ИИ уйдёт в `SKIP` навсегда. Сейчас стриминг выключен.
- **`BACKLOG.md` сильно отстал**: T2 помечен 🆕, хотя закрыт 01.07; нет фиксов инцидента 10.07,
  нет T3, нет задачи по дубликатам. Требует ревизии.

## Открытые задачи

1. **Мощность ИИ (решение владельца, деньги).** `AI_PROCESS_LIMIT=50` × 4 цикла
   (`CYCLE_INTERVAL_SECONDS=21600`) = **200 статей/сутки** против притока **~380/сутки**. Хвост
   (~6500) сам не рассосётся и растёт на ~180/день. Цена ~$0.018/статья: **~$120 разово** на хвост,
   **~$205/мес** на приток. Рычаги по возрастанию цены:
   **(а)** резать приток — 78% отказов = мусорные источники (в логах: РГУ Губкина шлёт спорт и
   парусные регаты, МФТИ — олимпиады и выпускные); **(б)** гейт на `gpt-5.4` вместо `5.5` (−50% на
   самой дорогой стадии); **(в)** поднять лимит до ~150 (600/сутки → хвост уйдёт за ~2 недели).
2. **Telegram недоступен с РФ-сервера**: `t.me → [Errno 101] Network is unreachable`. Раньше
   работал (см. память `oiltech-source-health`). Telegram-источники мертвы.
3. **Гос-сайты таймаутят**: `minpromtorg.gov.ru`, `rst.gov.ru` — 3 попытки × 20 с каждый, съедают
   время цикла `parse`.
4. **Дубликаты** (задача обсуждена, не начата): статус `duplicate` в бэкенде никто не пишет, плитка
   мертва, дедуп только по точному URL. `content_hash` **не годится** — в него зашит URL
   (`sha256(нормТайтл|нормURL)`), поэтому одна новость с разных доменов даёт разный хэш. Согласовано:
   near-dup детектор по **заголовку** (эмбеддинги `title_ru`+суть в окне ±3 дня + слои точного/нечёткого
   совпадения), сначала теневой прогон для подбора порогов. Открытый вопрос: прятать дубликат или
   схлопывать в группу «об этом написали ещё N источников».
5. **Гейт режет 78%** — стоит посмотреть отдельно: это качество корпуса, приоритет №1 заказчика по MVP2.
6. **Файлы прототипов 732–986 строк** при правиле «до 500» — разбить на компоненты.
7. Открытое из аудита: **T4** (счёт AI врёт ~100×), **T5** (nano-хвост ~1010 статей),
   **T7** (`/api/process` любому юзеру + открытая регистрация), **T8** (cookie без `Secure`).
8. **CI нет.** Инцидент 10.07 приехал из хотфикса, выкаченного в `main` с красными тестами.
   `docx`-тесты уже кодируют контракт — гейт на тесты закрыл бы весь класс.
