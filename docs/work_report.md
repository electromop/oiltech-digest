## Часть B. Доработки 2-го цикла (выполнены)

### B1. AI-фильтр релевантности — отдельный этап ✅
- Новый этап `relevance` в пайплайне: **суть → релевантность → теги → скоринг**.
- Бинарный AI-вердикт «релевантно нефтесервису?» + причина (`article_cards.relevant`, `relevance_reason`).
- Нерелевантные → `status='rejected'`, **автоматически исключаются из тегирования, скоринга и дайджеста** (`get_articles_needing_tags/scores`, `digest_candidates`).
- В UI — бейдж «нерелевантно» + причина в карточке.
- **Проверено на реальном GPT:** из 14 статей отсеяно 7 — культура (картина Лагорио), мясная индустрия (Черкизово), политика (Лавров, Лукашенко), страхование, геополитика. Нефтегаз (QatarEnergy, Газпромнефть) — прошёл.
- Файлы: `processing/prompts.py`, `processing/pipeline.py`, `processing/openai_client.py`, `db/repository.py`, `cli.py`, `api.py`.

### B2. Детекция неполного текста RSS ✅
- Эвристика `is_truncated` (пустой / концовка-многоточие / «read more»/«читать далее» / < 280 симв.).
- Флаг `articles.text_truncated`, проставляется при `parse`, backfill всех 807 статей.
- **Результат: 568 из 807 (70%) помечены неполными** — подтверждает проблему сокращённых лент.
- В UI — бейдж «неполный текст» на карточке. Отдаётся в `/api/articles`.
- Файлы: `ingestion/normalize.py`, `ingestion/rss_parser.py`, `db/schema.sql`, `db/repository.py`, `api.py`, `web/app.html`.

### B3. Экраны «Скоринг» и «Источники» (как в мокапе) + Docker ✅
- **Скоринг** — редактируемый 1-в-1 как мокап: правка названия/веса/описания/ключей, «Добавить», «Удалить», «Нормализовать», «Сохранить» (доступно только при Σ=100%; бэкенд тоже валидирует → HTTP 400). Новые `PUT/DELETE /api/scoring-criteria`, `repository.save_scoring_criteria`.
- **Источники** — переключатель вкл/выкл и правка RSS прямо в карточке (`PATCH /api/sources`).
- **Docker** — `Dockerfile` + сервис `app` в `docker-compose`; приложение собрано и поднято в контейнере (`oiltech_app` ↔ `oiltech_pg`), `/api/health` отвечает из контейнера.

## Часть D. Стоимость по каждой функции (задача #10)

Модель `gpt-5-nano`, цены: вход $0.05 / выход $0.40 за 1M токенов. Визуально — в [`docs/cost_dashboard.html`](cost_dashboard.html).

| Функция | Операций | Ср. токенов | $ / операцию | $ / 1000 операций |
|---------|---------:|------------:|-------------:|------------------:|
| Суть (summary) | 14 | 634 | $0.0000867 | $0.087 |
| Релевантность | 14 | 830 | $0.0000701 | $0.070 |
| Тегирование | 7 | 2 178 | $0.0001396 | $0.140 |
| Скоринг | 7 | 1 546 | $0.0002023 | $0.202 |
| **Полный цикл 1 статьи (4 этапа)** | — | ≈4 200 | **≈$0.00050** | **$0.50** |

**Разница RU vs EN (средние токены на операцию):**

| Функция | EN | RU |
|---------|---:|---:|
| Суть | 1 426 | 417 |
| Релевантность | 1 668 | 601 |
| Тегирование | 2 764 | 1 738 |
| Скоринг | 2 200 | 1 055 |

- EN-операции в выборке дороже по токенам — это длинные ленты (TechCrunch/MIT). Разница частично от длины контента; выборка небольшая → цифры **индикативные**.
- **Тегирование** — больше всего входных токенов (в промпт уходит список 18 тегов с ключами); **скоринг** — дороже всех по выходу (объяснения по критериям); **суть/релевантность** — самые дешёвые.
- **Экономия от фильтра:** отклонённая статья (~$0.00016) втрое дешевле релевантной (~$0.00050) — не доходит до тегов/скоринга. На потоке 1000 статей при 50% релевантности: **≈$0.33** против ≈$0.50 без фильтра (~34% экономии).
- **«Перевод» тегов** не тратит токены на статью — EN берётся из Excel при `seed-tags`.

## Заметки и риски
