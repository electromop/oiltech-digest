## 1) Корень «замолчавших» источников

**Не сеть и не селекторы — неверный URL.** У массы источников прописан адрес, который не является
лентой новостей. Парсер честно его скребёт, находит навигацию, один раз кладёт её как «статьи», дальше
получает только дубликаты. Внешне — «источник замолчал».

| Источник | Что было по URL |
|---|---|
| ДРТ | `rss.xml` = **карта сайта** («HR Portal», «Благотворительный фонд»); парс: `добавлено=0, дублей=290` |
| Petronas | `rss.xml` = страница **наград** |
| Сургутнефтегаз | без `listing_url` скреб **главную** («История дивидендных выплат», «Банковские реквизиты») |

**Пре-фильтр это не ловит структурно:** служебная страница нефтяной компании содержит слово «нефть»
(это её название) и проходит как профильная.

**Как читать диагностику.** `last_parsed_at` — дата **последнего успешного** фетча (`touch_last_parsed`
стоит после раннего выхода при `content is None`). Значит:
- `парсили = сегодня` + 0 статей → **неверный URL**;
- `парсили = месяц назад` → **фетч падает**.

**Сделано (PR #29):** 19 правок в `source_overrides` (29 → 48 записей). Деплой:
`Оверрайды: изменено=27, без изменений=21, не найдено=0`.
**`не найдено=0` — обязательная проверка:** реестр ключуется точным `sources.name`, опечатка означает,
что правка молча не применится.

**Добавлены тесты `apply_overrides`** (их не было вовсе, а реестр применяется на каждом деплое): не
трогает статьи, сбрасывает дедуп-состояние, идемпотентен, гигиена реестра. Первый же тест нашёл дыру:
`source_overrides` импортирует `get_connection` напрямую, conftest его не патчил → **тесты ходили в
настоящую базу**.

## 2) Системный баг парсера — срезался финальный слэш (PR #30)

Найден **живым парсом сразу после деплоя** — проверка себя окупила.

```
в листинге: /press-center/press_releases/preduprezhdenie-.../   ← со слэшем
запрашивал:  /press-center/press_releases/preduprezhdenie-...    ← без → 404
```

`request_parser` резал слэш (`parts.path.rstrip("/")`), а корпоративные РФ-сайты на Bitrix без него
отдают 404. Срез был нужен **только** чтобы отсеять главную → оставлен в проверке, убран из URL.

**Бласт-радиус сужен намеренно:** корень `/` по-прежнему схлопывается в пустой путь. Иначе query-only
ссылки сменили бы форму (`site?p=678` → `site/?p=678`), а `articles.url` уникален → уже сохранённые
статьи вставились бы **заново как дубликаты**. Это поймал существующий тест — фикс сузили, тест не трогали.

  Но и `source-health` говорит только «молчит», а **почему** — показывает `source-diagnose`.

  рендером (200, свежие даты 06/07/2026). Проверять перед тем, как опираться.
