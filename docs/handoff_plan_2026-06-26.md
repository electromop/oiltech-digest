### C. Перепрогон релевантности + мягкое удаление
- Гейт релевантности ужесточён (явные исключения: война/СВО, криминал, ДТП, общая политика/макро,

- Перепрогон всей базы новой моделью **gpt-5.5** (gate-only по сырому тексту):
  `enqueue-recheck` (батчи через NL-воркер), `recheck-relevance` (локально).
- **Мягкое удаление (safety):** `enqueue-recheck --mark` помечает нерелевантные
  (`articles.pending_deletion`), они исчезают из ленты, но физически в БД. Просмотр —
  `recheck-marked`; разовое удаление — `recheck-purge --yes`; возврат — `recheck-unmark`.
- Диагностика: `enqueue-recheck --dry-run` + `recheck-dry-show <job_id>` — увидеть вердикты
  гейта без удаления (выборка случайная по базе).
- Фикс регрессии: `set_article_relevance` → UPSERT; «Сигналы» прячут `relevant=false`.

## 2. Текущее состояние прода

- **Архитектура (живая):** РФ-core (СПб) — БД + UI + домен + парсинг; зарубежный
  stateless-воркер (Амстердам) — OpenAI по HTTPS task-API (без БД/VPN).
- **Модели:** gpt-5.5 (гейт релевантности) + gpt-5-mini (суть/теги/скоринг/перевод).
- **Источники:** 120 (request 56 · rss 31 · playwright 18 · telegram 15), рабочих ~71.
- **Деплой сервера:** `git fetch && git reset --hard origin/<ветка>` (НЕ `git pull` —
  ловушка divergent branches), затем `docker compose up -d --build`.

### ⚠️ Незакрытое по перепрогону релевантности (ВАЖНО)
- Реальный recheck (физическое удаление) успел срезать базу **7643 → ~5400**
  (≈2000 статей удалено **безвозвратно**), затем был **остановлен** (очереди recheck и
  translate отменены) — пользователь счёл отсев слишком агрессивным (~60%).
- В ответ построено **мягкое удаление** (`--mark`/`recheck-marked`/`recheck-purge`/
  `recheck-unmark`) — запушено, **НЕ задеплоено**.
- **План продолжения:** деплой → `enqueue-recheck --mark` (пометить, не удаляя) →
  `recheck-marked` (глазами проверить, что помечен мусор) → `recheck-purge --yes` (если ок)
  ИЛИ `recheck-unmark` + смягчить промпт гейта (3 правки: «сомнение→оставлять край»,
  источник как слабый сигнал, не резать по одному заголовку при пустом тексте) →
  вернуть `enqueue-translate`.
